# Trace MVP 명세서

## MVP 목표

Trace MVP의 목표는 Trace 자체 캡처 오버레이로 빠르게 영역을 캡처하거나 전체 화면을 캡처하고, 저장하고, 복사만 하기 또는 앱으로 자동 전달하기 중 하나의 캡처 방식을 사용할 수 있게 하며, 최근 및 과거 캡처 이력을 확인할 수 있는 최소 제품을 만드는 것이다.

AI 기반 자동 분류, 자동 파일명 생성, 내용 분석, OCR, 스마트 검색은 MVP에 포함하지 않는다. MVP에서는 자체 영역 선택 오버레이, 전체 화면 캡처, 캡처 결과 생성을 구현하고, 모든 캡처를 날짜별 폴더에 저장하며, 앱 내부 히스토리에서 시간순으로 확인하는 데 집중한다.

## MVP 포함 기능

### 1. 빠른 캡처 시작

사용자는 macOS 어디에서든 Trace 캡처를 빠르게 시작할 수 있다. MVP에는 복사만 하기 모드와 앱으로 자동 전달 모드가 있다.

기본 요구사항:

- 앱이 백그라운드에 있어도 단축키가 동작한다.
- 단축키 입력 시 캡처 오버레이가 바로 열린다.
- 사용자는 캡처 오버레이가 열린 상태에서 영역 캡처, 전체 화면 캡처, 복사만 하기, 앱으로 자동 전달을 바꿀 수 있다.
- 영역 캡처에서는 사용자가 Trace 오버레이에서 드래그로 캡처 영역을 지정한 뒤 캡처 버튼, Enter, Space, Command-C 중 하나로 확정한다.
- 전체 화면 캡처에서는 사용자가 오버레이의 캡처 버튼, Enter, Space, Command-C 중 하나로 현재 디스플레이 전체를 캡처한다.
- 캡처 취소가 가능해야 한다.
- 사용자는 복사만 하기 캡처와 앱으로 자동 전달 캡처 중 하나를 사용할 수 있다.

### 2. 캡처 범위

사용자는 Trace 자체 오버레이로 화면의 원하는 영역을 선택해 이미지로 캡처하거나, 현재 디스플레이 전체 화면을 캡처한다.

기본 요구사항:

- MVP에서는 Trace가 자체 영역 선택 오버레이를 구현한다.
- Trace는 선택 영역 좌표를 계산하고, 해당 영역의 화면 이미지를 생성한다.
- 전체 화면 캡처를 선택하면 영역 드래그 없이 오버레이가 표시된 디스플레이 전체 이미지를 생성한다.
- 선택한 범위 또는 전체 화면 이미지만 저장한다.
- 캡처 완료 직후 이미지가 클립보드에 복사된다.
- 사용자가 Trace 오버레이에서 취소하면 저장, 복사, 전달을 진행하지 않는다.
- 저장 실패와 복사 실패는 사용자에게 알려야 한다.

### 3. 캡처 방식

캡처 방식은 두 가지 모드로 나뉜다.

#### 복사만 하기 모드

캡처 범위를 선택하면 이미지가 저장되고 클립보드에 자동 복사된다. 대상 앱 선택 창은 띄우지 않는다.

기본 요구사항:

- 캡처 이미지를 날짜별 폴더에 저장한다.
- 캡처 이미지를 클립보드에 자동 복사한다.
- 저장 완료 위치를 사용자에게 알려준다.
- 사용자는 원하는 앱에 직접 붙여넣을 수 있다.

#### 앱으로 자동 전달 모드

자동 전달 캡처 상태에서 사용자가 캡처 범위를 선택하면 이미지가 저장되고 클립보드에 자동 복사된다. 그 다음 Trace가 현재 실행 중인 앱 목록을 보여주고, 사용자가 전달 대상을 선택하면 Trace가 해당 앱을 활성화한 뒤 붙여넣기 이벤트를 보내 이미지가 들어가게 한다.

기본 요구사항:

- 캡처 이미지를 날짜별 폴더에 저장한다.
- 캡처 이미지를 클립보드에 자동 복사한다.
- 캡처 범위 선택이 끝난 뒤 전달 대상 선택 창을 표시한다.
- 현재 실행 중인 앱 목록을 보여준다.
- 선택한 앱의 윈도우 목록을 보여준다.
- 선택한 윈도우를 활성화하고 붙여넣기 이벤트를 보내 캡처 이미지 전달을 시도한다.
- 대상 앱이 붙여넣기를 받을 수 없거나 macOS 권한이 없으면 실패 상태를 알려야 한다.
- 사용자는 대상 앱 선택을 건너뛸 수 있다.

### 4. 날짜별 자동 저장

캡처한 이미지는 사용자가 지정한 기본 저장 위치 아래에 날짜별 폴더로 저장된다.

저장 구조 예시:

```text
Trace/
  captures/
    2026-05-17/
      2026-05-17_14-32-08.png
      2026-05-17_14-35-21.png
    2026-05-18/
      2026-05-18_09-10-04.png
  thumbnails/
    2026-05-17/
      2026-05-17_14-32-08.jpg
  metadata.json
  settings.json
```

기본 요구사항:

- 최초 실행 시 기본 저장 위치를 설정한다.
- 날짜별 폴더는 자동 생성한다.
- 파일명은 캡처 시각 기반으로 생성한다.
- 같은 시각에 중복 파일명이 생기면 충돌을 피한다.
- 저장 완료 후 저장 위치를 사용자에게 알려준다.

### 5. 최근 캡처 보기

사용자는 최근 저장한 캡처를 빠르게 확인할 수 있다.

기본 요구사항:

- 최근 캡처 목록을 시간 역순으로 보여준다.
- 썸네일, 저장 시각, 파일 위치를 표시한다.
- 항목을 클릭하면 원본 이미지를 미리 볼 수 있다.
- 최근 항목에서 이미지를 다시 클립보드에 복사할 수 있다.

### 6. 과거 캡처 이력 보기

사용자는 날짜별로 과거 캡처 이력을 확인할 수 있다.

기본 요구사항:

- 날짜별 그룹으로 캡처 이력을 보여준다.
- 각 캡처의 썸네일과 저장 시간을 보여준다.
- 캡처 파일을 Finder에서 열 수 있다.
- 캡처 이미지를 다시 클립보드에 복사할 수 있다.

## MVP 제외 기능

다음 기능은 MVP에서 만들지 않는다.

- AI 기반 캡처 내용 분석
- AI 기반 자동 폴더 분류
- AI 기반 자동 파일명 생성
- OCR 텍스트 추출
- 캡처 내용 검색
- 이미지 주석, 하이라이트, 모자이크
- 클라우드 동기화
- 팀 공유
- 앱별 커스텀 전달 규칙
- 자주 쓰는 앱 자동 추천

## 기본 사용자 흐름

### 복사만 하기 캡처 흐름

1. 사용자가 전역 단축키를 누른다.
2. Trace 캡처 오버레이가 열린다.
3. 사용자가 오버레이 안에서 복사만 하기와 캡처 범위를 확인하거나 변경한다.
4. 영역 캡처이면 사용자가 영역을 드래그해 선택한 뒤 오버레이의 캡처 버튼, Enter, Space, Command-C 중 하나로 확정하고, 전체 화면 캡처이면 같은 입력으로 캡처한다.
5. Trace가 이미지를 저장한다.
6. Trace가 이미지를 클립보드에 복사한다.
7. 저장 위치 알림이 표시된다.
8. 대상 앱 선택 창은 표시되지 않는다.

### 앱으로 자동 전달 캡처 흐름

1. 사용자가 전역 단축키를 누른다.
2. Trace 캡처 오버레이가 열린다.
3. 사용자가 오버레이 안에서 앱으로 자동 전달과 캡처 범위를 확인하거나 변경한다.
4. 영역 캡처이면 사용자가 영역을 드래그해 선택한 뒤 오버레이의 캡처 버튼, Enter, Space, Command-C 중 하나로 확정하고, 전체 화면 캡처이면 같은 입력으로 캡처한다.
5. Trace가 이미지를 저장하고 클립보드에 복사한다.
6. Trace가 현재 실행 중인 앱 목록을 보여준다.
7. 사용자가 전달할 앱을 선택한다.
8. Trace가 선택한 앱을 활성화하고 붙여넣기 이벤트를 보낸다.
9. 성공, 실패, 또는 건너뛰기 상태가 히스토리에 기록된다.

### 과거 캡처를 다시 사용하는 흐름

1. 사용자가 Trace 히스토리 창을 연다.
2. 최근 캡처 또는 날짜별 이력을 확인한다.
3. 원하는 캡처를 선택한다.
4. 이미지를 미리 보거나 Finder에서 연다.
5. 필요하면 다시 클립보드에 복사한다.

## 화면 구성

### 메뉴바 또는 백그라운드 앱

MVP에서는 Trace가 항상 접근 가능해야 한다. macOS 메뉴바 앱 형태가 적합하다.

필요 기능:

- 캡처 시작
- 캡처 중 범위와 방식 변경
- 최근 캡처 열기
- 히스토리 열기
- 설정 열기
- 앱 종료

### 캡처 오버레이

화면 위에 Trace 자체 오버레이를 띄우고, 사용자가 드래그로 영역을 선택한다.

필요 기능:

- 캡처 범위 전환
- 복사만 하기와 앱으로 자동 전달 전환
- 드래그 선택
- 선택 영역 시각화
- 취소
- 캡처 완료
- 캡처 버튼, Enter, Space, Command-C로 캡처 확정
- 멀티 디스플레이와 Retina 좌표 처리
- 너무 작은 선택 영역 무시

### 전달 대상 선택 창

앱으로 자동 전달 모드에서 캡처 범위 선택이 끝난 직후 작고 빠른 선택 창을 띄운다.

필요 기능:

- 실행 중인 앱 목록
- 선택한 앱의 윈도우 목록
- 앱 아이콘과 이름 표시
- 선택한 윈도우 활성화와 붙여넣기 이벤트 전송
- 건너뛰기

### 히스토리 창

최근 캡처와 과거 캡처를 확인하는 메인 관리 화면이다.

필요 기능:

- 최근 캡처 섹션
- 날짜별 캡처 목록
- 썸네일 그리드
- 이미지 미리보기
- 클립보드 복사
- Finder에서 보기

### 설정 창

사용자가 기본 동작을 조정하는 화면이다.

필요 기능:

- 저장 위치 선택
- 전역 단축키 설정
- 기본 캡처 방식 설정
- macOS 알림 설정 열기

## 알림 정책

캡처 저장 후 사용자가 저장 위치를 알 수 있어야 한다.

MVP 알림 예시:

- 제목: `캡처 저장 완료`
- 내용: `2026-05-17 폴더에 저장됨`
- 액션: Finder에서 보기

알림은 너무 방해가 되지 않아야 하며, 켜기/끄기는 macOS 시스템 알림 설정에서 관리한다. Trace 설정은 현재 상태를 보여주고 해당 시스템 설정으로 이동할 수 있어야 한다.

## 권한 정책

Trace는 macOS 시스템 기능을 사용하므로 권한 상태를 사용자가 이해할 수 있게 보여줘야 한다. 자세한 정책은 `docs/development/permissions.md`를 따른다.

MVP에서 고려할 권한:

- 화면 캡처를 위한 Screen Recording 권한
- 앱 활성화와 붙여넣기 이벤트 전송을 위한 Accessibility 권한
- 저장 완료 알림을 위한 Notifications 권한

필수 권한이 없으면 해당 기능을 실행하지 않고, 설정 화면 또는 권한 안내 화면에서 필요한 조치를 알려준다.

## 성공 기준

MVP가 성공했다고 판단할 수 있는 기준은 다음과 같다.

- 사용자가 단축키로 2초 이내에 캡처를 시작할 수 있다.
- 캡처 후 이미지가 자동 저장되고 클립보드에 복사된다.
- 사용자가 복사만 하기 캡처와 앱으로 자동 전달 캡처 중 하나를 사용할 수 있다.
- 앱으로 자동 전달 캡처에서는 범위 선택 후 전달 대상 앱을 선택할 수 있다.
- 사용자가 전달 대상 앱을 선택하면 대표 대상 앱에서 붙여넣기 기반 자동 전달이 동작한다.
- 최근 캡처를 앱 안에서 다시 찾을 수 있다.
- 날짜별 과거 이력을 확인할 수 있다.
- AI 기능 없이도 기본 캡처 도구보다 반복 작업이 줄어든다.

## 후속 버전 후보

- AI 기반 자동 파일명 생성

- OCR 기반 텍스트 검색
- 캡처 이미지 주석 도구
- 민감 정보 자동 블러
- 클라우드 백업
