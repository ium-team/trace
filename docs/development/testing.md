# 테스트 전략

## 목표

Trace의 MVP 테스트는 캡처 앱의 핵심 흐름이 반복 사용 중 깨지지 않는 것을 목표로 한다.

## 단위 테스트 대상

### 저장 경로

- 날짜별 폴더 경로 생성
- 파일명 생성
- 파일명 충돌 처리
- 상대 경로와 절대 경로 변환
- 원자적 메타데이터 쓰기
- 손상된 메타데이터 파일 복구

### 설정

- 기본 설정 생성
- 설정 저장
- 설정 불러오기
- 잘못된 설정 파일 처리

### 메타데이터

- 캡처 항목 추가
- 최신순 정렬
- 날짜별 그룹화
- 누락 파일 상태 표시

### 캡처 방식 분기

- 복사만 하기 모드에서는 전달 대상 선택 창을 띄우지 않는다.
- 앱으로 자동 전달 모드에서는 캡처 후 전달 대상 선택 창을 띄운다.
- 건너뛰기 시 저장과 클립보드 복사는 유지된다.
- Accessibility 권한이 없으면 자동 전달을 시도하지 않는다.
- 대상 앱 전달 실패 시 실패 상태가 기록된다.

### 권한 상태

- Screen Recording 권한이 없거나 macOS 기본 캡처 도구 실행이 실패하면 캡처 시작이 권한 또는 실패 안내로 이어진다.
- Accessibility 권한이 없으면 앱 자동 전달이 권한 안내로 이어진다.
- Notifications 권한이 없으면 저장은 성공하고 알림만 생략된다.

## 통합 테스트 대상

### 복사만 하기 캡처

1. 캡처 시작
2. macOS 기본 캡처 도구에서 영역 선택
3. 이미지 저장
4. 클립보드 복사
5. 히스토리 반영

### 앱으로 자동 전달 캡처

1. 자동 전달 캡처 시작
2. macOS 기본 캡처 도구에서 영역 선택
3. 이미지 저장
4. 클립보드 복사
5. 전달 대상 선택 창 표시
6. 앱 선택
7. 앱 활성화와 붙여넣기 이벤트 전송 완료
8. 히스토리 반영

### 히스토리 재사용

1. 히스토리 창 열기
2. 최근 캡처 확인
3. 날짜별 이력 확인
4. 이미지 미리보기
5. 클립보드에 다시 복사
6. Finder에서 보기

## 수동 점검 체크리스트

- 메뉴바 앱이 실행되는가
- 단축키로 캡처가 시작되는가
- 캡처 취소가 가능한가
- macOS 기본 캡처 도구에서 선택한 영역의 결과 파일을 받을 수 있는가
- 저장 위치에 날짜별 폴더가 생성되는가
- 클립보드에 이미지가 들어가는가
- 복사만 하기 모드에서 전달 대상 창이 뜨지 않는가
- 자동 전달 모드에서 범위 선택 후 전달 대상 창이 뜨는가
- 자동 전달 실패 시 사용자가 실패를 알 수 있는가
- 히스토리에서 최근 캡처가 최신순으로 보이는가
- 파일을 Finder에서 열 수 있는가
