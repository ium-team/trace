# 구현 계획

## 1단계: 프로젝트 기반

목표:

- macOS 앱 프로젝트를 만든다.
- 메뉴바 앱으로 실행되게 한다.
- 기본 설정 저장 구조를 만든다.

완료 기준:

- 앱 실행 시 메뉴바 아이콘이 보인다.
- 메뉴에서 설정과 히스토리 창을 열 수 있다.
- 설정 파일을 읽고 쓸 수 있다.

## 2단계: 캡처 오버레이

목표:

- 사용자가 화면에서 영역을 선택할 수 있게 한다.

완료 기준:

- 캡처 시작 시 화면 오버레이가 표시된다.
- 드래그 영역이 시각적으로 표시된다.
- 취소가 가능하다.
- 선택 영역 좌표를 얻을 수 있다.

## 3단계: 캡처 저장과 클립보드

목표:

- 선택 영역을 이미지로 저장하고, 자동 복사 설정이 켜져 있으면 클립보드에 복사한다.

완료 기준:

- PNG 파일이 날짜별 폴더에 저장된다.
- 파일명 충돌이 처리된다.
- 자동 복사 설정이 켜져 있으면 클립보드에 이미지가 복사된다.
- 저장 완료 알림이 표시된다.

## 4단계: 기본 캡처

목표:

- 가장 단순한 캡처 흐름을 완성한다.

완료 기준:

- 사용자가 단축키로 기본 캡처를 시작한다.
- 범위를 선택하면 필요하면 클립보드 복사가 완료된다.
- 저장 옵션이면 이미지 저장도 완료된다.
- 전달 대상 선택 창은 표시되지 않는다.

## 5단계: 앱으로 자동 전달 캡처

목표:

- 자동 전달 캡처 상태에서 범위 선택 후 전달 대상 앱을 고르고 붙여넣기 기반 전달을 시도하게 한다.

완료 기준:

- 사용자가 앱으로 자동 전달 캡처를 시작한다.
- 범위를 선택하면 전달 대상 선택 창이 열린다.
- 실행 중인 앱 목록이 표시된다.
- 사용자가 앱을 선택하면 앱 활성화와 붙여넣기 이벤트 전송이 수행된다.
- 건너뛰기를 선택하면 전달용 클립보드는 유지되고, 저장 옵션이면 저장도 유지된다.

## 6단계: 히스토리

목표:

- 최근 캡처와 날짜별 캡처 이력을 볼 수 있게 한다.

완료 기준:

- 최근 캡처가 시간 역순으로 표시된다.
- 날짜별 캡처 이력이 표시된다.
- 이미지를 미리 볼 수 있다.
- 다시 클립보드에 복사할 수 있다.
- Finder에서 파일 위치를 열 수 있다.

## 7단계: 안정화

목표:

- 반복 사용 가능한 수준으로 다듬는다.

완료 기준:

- 저장 실패가 사용자에게 표시된다.
- 클립보드 복사 실패가 사용자에게 표시된다.
- 자동 전달 실패가 사용자에게 표시된다.
- 캡처 취소와 건너뛰기 흐름이 자연스럽다.
- 핵심 흐름 테스트가 통과한다.
