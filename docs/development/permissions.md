# macOS 권한 정책

## 목표

Trace는 화면 캡처, 전역 접근, 앱 자동 전달을 위해 macOS 시스템 권한에 의존한다. MVP에서는 권한이 없을 때 기능이 조용히 실패하지 않게 하고, 사용자가 필요한 조치를 이해할 수 있게 한다.

## 권한 목록

### Screen Recording

필요한 기능:

- 화면 이미지 확보
- 영역 선택 후 실제 캡처 이미지 생성

정책:

- 캡처 시작 전에 권한 상태를 확인한다.
- 권한이 없으면 캡처 오버레이를 열지 않고 권한 안내를 표시한다.
- 안내에는 시스템 설정으로 이동하는 액션을 제공한다.
- 사용자가 권한을 부여한 뒤 앱 재시작이 필요하면 재시작 필요 상태를 표시한다.

### Accessibility

필요한 기능:

- 선택한 앱 활성화
- 붙여넣기 이벤트 전송

정책:

- 복사만 하기 모드에는 필수 권한으로 보지 않는다.
- 앱으로 자동 전달 모드에서만 필수 권한으로 다룬다.
- 권한이 없으면 자동 전달을 시도하지 않고, 저장과 클립보드 복사까지만 완료한다.
- 권한 부족으로 전달하지 못한 캡처는 `deliveryState`를 `failed`로 기록한다.

### Notifications

필요한 기능:

- 저장 완료 알림
- 실패 알림

정책:

- 알림 권한은 캡처 성공의 필수 조건이 아니다.
- 권한이 없으면 캡처와 저장은 정상 진행하고 알림만 생략한다.
- 설정에서 알림 표시 여부를 끄면 권한 요청을 하지 않아도 된다.

## 권한 안내 UI

권한 안내는 설정 창 또는 작은 안내 창에서 제공한다.

필수 요소:

- 필요한 권한 이름
- 권한이 필요한 이유
- 시스템 설정 열기 액션
- 다시 확인 액션

## 실패 상태

권한 문제로 실패한 기능은 사용자에게 원인을 알려야 한다.

상태 예시:

- Screen Recording 없음: `화면 캡처 권한이 필요합니다.`
- Accessibility 없음: `앱으로 자동 전달하려면 손쉬운 사용 권한이 필요합니다.`
- Notifications 없음: 저장은 성공하고 알림은 표시하지 않는다.

## 구현 우선순위

1. 권한 상태 확인 유틸리티
2. 캡처 시작 전 Screen Recording 검사
3. 자동 전달 전 Accessibility 검사
4. 설정 창의 권한 상태 표시
5. 권한 실패 상태 테스트
