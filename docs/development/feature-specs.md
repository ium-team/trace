# 기능별 개발 명세

## 1. 앱 셸

### 목표

Trace가 메뉴바 앱으로 항상 실행되고, 사용자가 빠르게 캡처와 히스토리에 접근할 수 있게 한다.

### 요구사항

- 메뉴바 아이콘을 표시한다.
- 메뉴에서 캡처 시작, 히스토리 열기, 설정 열기, 종료를 제공한다.
- 앱 실행 중 전역 단축키가 동작한다.
- 앱 재실행 시 이전 설정을 불러온다.

## 2. 캡처 시작

### 목표

사용자가 어디서든 빠르게 캡처를 시작할 수 있게 한다.

### 요구사항

- 기본 캡처를 시작할 수 있다.
- 앱으로 자동 전달 캡처를 시작할 수 있다.
- 기본 캡처 단축키와 앱으로 자동 전달 캡처 단축키는 각각 설정할 수 있다.
- 두 단축키는 같은 오버레이를 열며, 시작 시 선택되는 캡처 방식만 다르다.
- 캡처 시작 시 화면 위에 Trace 자체 캡처 오버레이를 바로 표시한다.
- 캡처 오버레이가 열린 동안 포인터는 캡처용 십자 커서로 표시한다.
- 사용자는 오버레이가 열린 상태에서 영역 캡처, 전체 화면 캡처, 기본 캡처, 앱으로 자동 전달을 바꿀 수 있다.
- 영역 캡처 선택 시 오버레이에서 드래그로 캡처 범위를 선택한 뒤 캡처 버튼, Enter, Space, Command-C 중 하나로 확정한다.
- 전체 화면 캡처 선택 시 오버레이의 캡처 버튼, Enter, Space, Command-C 중 하나로 현재 디스플레이 전체를 캡처한다.
- 영역 캡처에서는 사용자가 드래그로 캡처 범위를 선택하고 캡처 버튼 또는 키보드 확정 입력으로 완료한다.
- 영역 캡처에서는 사용자가 캡처를 취소할 수 있다.
- Screen Recording 권한이 없으면 캡처 UI를 열기 전에 macOS 권한 요청을 트리거하고 캡처를 중단한다.
- 멀티 디스플레이 환경에서도 선택 영역 좌표가 올바르게 계산되어야 한다.
- Retina 디스플레이에서는 논리 좌표와 픽셀 좌표 변환을 처리해야 한다.
- 0 크기 또는 너무 작은 선택 영역은 캡처하지 않고 취소로 처리한다.

## 3. 기본 캡처

### 목표

캡처 이미지를 설정에 따라 저장하고, 자동 복사 설정이 켜져 있으면 클립보드에 복사한 뒤 흐름을 끝낸다.

### 요구사항

- 캡처 범위를 선택하거나 전체 화면 캡처를 시작하면 이미지가 생성된다.
- 자동 복사 설정이 켜져 있으면 이미지는 클립보드에 자동 복사된다.
- 설정이 `저장`이면 이미지는 날짜별 폴더에 저장되고 저장 완료 위치를 사용자에게 알려준다.
- 설정이 `저장 안 함`이면 이미지는 저장하지 않고 히스토리에 기록하지 않는다.
- 전달 대상 선택 창은 표시하지 않는다.

## 4. 앱으로 자동 전달 캡처

### 목표

자동 전달 캡처 상태에서 범위를 선택한 뒤, 사용자가 전달 대상 앱을 고르면 해당 앱을 활성화하고 붙여넣기 이벤트를 보내게 한다.

### 요구사항

- 자동 전달 캡처 상태는 오버레이 안에서 선택하거나 변경할 수 있다.
- 사용자가 캡처 범위를 선택하거나 전체 화면 캡처 버튼을 누르면 이미지가 생성된다.
- 이미지는 붙여넣기 전달을 위해 클립보드에도 준비된다.
- 설정이 `전달`이면 저장하지 않고 전달만 수행한다.
- 설정이 `저장 및 전달`이면 이미지를 저장하고 전달 상태를 히스토리에 기록한다.
- 전달 대상 방식이 `매번 앱 선택`이면 캡처 범위 선택 직후 전달 대상 선택 창을 표시한다.
- 전달 대상 선택 창에는 현재 실행 중인 앱 목록을 표시한다.
- 사용자가 앱을 선택하면 해당 앱의 윈도우 목록을 표시한다.
- 앱이 Accessibility 윈도우 목록을 충분히 노출하지 않아도, 화면에 실제로 떠 있는 일반 윈도우는 선택지에 표시해야 한다.
- 현재 Space가 아닌 다른 Space의 일반 윈도우도 선택지에 표시해야 한다.
- 윈도우 선택지에는 가능한 경우 실제 윈도우 썸네일을 함께 표시해야 한다.
- 사용자가 윈도우를 선택하면 해당 윈도우를 활성화한 뒤 붙여넣기 이벤트를 보낸다.
- 전달 대상 방식이 `가장 최근 사용 앱`이면 캡처 시작 직전에 활성화되어 있던 앱을 선택 창 없이 활성화하고 붙여넣기 이벤트를 보낸다.
- 전달 대상 방식이 `지정한 앱`이면 설정에 저장된 앱으로 바로 전달한다.
- 지정 앱이 실행 중이지 않으면 실패 알림을 보내고 자동 실행은 하지 않는다.
- 지정 앱의 윈도우가 여러 개일 수 있으므로, 설정에서 `가장 최근 사용 윈도우` 또는 `윈도우 선택` 중 하나를 고를 수 있어야 한다.
- 특정 앱은 앱별 전달 어댑터를 통해 윈도우보다 더 깊은 대상을 추가로 제공할 수 있어야 한다.
- Accessibility 권한이 없으면 macOS 권한 요청을 트리거하고 자동 전달을 시도하지 않는다.
- 대상 앱이 붙여넣기를 받을 수 없으면 실패 상태를 표시하고, 저장된 캡처가 있으면 히스토리에 기록한다.
- 사용자는 전달 대상 선택을 건너뛸 수 있다.
- 건너뛰어도 전달용 클립보드는 유지되고, 저장 옵션이면 저장도 유지된다.

## 5. 날짜별 저장

### 목표

캡처 이미지가 예측 가능한 위치에 날짜별로 정리되게 한다.

### 요구사항

- 기본 저장 폴더 아래 `captures/YYYY-MM-DD` 폴더를 만든다.
- 썸네일은 `thumbnails/YYYY-MM-DD` 폴더에 저장한다.
- 폴더명은 `YYYY-MM-DD` 형식을 사용한다.
- 파일명은 설정에서 선택한 규칙을 사용한다.
- 날짜/시간 규칙에서는 `YYYY-MM-DD_HH-mm-ss`, `YYYY-MM-DD_HH-mm`, `YYYY-MM-DD` 중 하나를 사용할 수 있다.
- 순서 규칙에서는 `001, 002, 003` 또는 `가, 나, 다` 스타일로 파일명을 생성할 수 있다.
- 같은 이름이 이미 있으면 충돌 없이 저장한다.
- 메타데이터는 원자적으로 쓰고, 쓰기 실패 시 원본 파일 저장 성공 여부와 분리해 처리한다.
- 저장 완료 후 히스토리에 표시할 수 있어야 한다.

## 6. 최근 캡처

### 목표

사용자가 방금 캡처한 이미지를 빠르게 다시 확인하고 재사용할 수 있게 한다.

### 요구사항

- 최근 캡처를 시간 역순으로 표시한다.
- 각 항목은 썸네일, 저장 시각, 파일 위치를 보여준다.
- 사용자는 이미지를 미리 볼 수 있다.
- 사용자는 이미지를 다시 클립보드에 복사할 수 있다.
- 사용자는 Finder에서 파일 위치를 열 수 있다.
- 사용자는 캡처 이름을 편집할 수 있다.
- 사용자는 캡처를 삭제할 수 있다.
- 사용자는 중요한 캡처를 고정하거나 북마크할 수 있다.

## 7. 과거 캡처 이력

### 목표

날짜별로 쌓인 캡처를 앱 안에서 찾아볼 수 있게 한다.

### 요구사항

- 날짜별 그룹으로 캡처 이력을 표시한다.
- 각 날짜 그룹은 최신 날짜가 위에 온다.
- 같은 날짜 안에서는 최신 캡처가 위에 온다.
- 썸네일 로딩은 UI를 막지 않아야 한다.
- 파일이 사라진 항목은 사용자에게 구분되어야 한다.
- 고정된 캡처는 별도 고정 섹션에도 표시한다.

## 8. 설정

### 목표

사용자가 Trace의 기본 동작을 조정할 수 있게 한다.

### 요구사항

- 저장 위치를 설정할 수 있다.
- 저장 파일명 규칙(날짜/시간 또는 순서)을 설정할 수 있다.
- 날짜/시간 규칙일 때 날짜 형식을 설정할 수 있다.
- 순서 규칙일 때 순서 형식을 `가, 나, 다` 또는 `001, 002, 003`로 설정할 수 있다.
- 기본 캡처 전역 단축키를 설정할 수 있다.
- 앱으로 자동 전달 캡처 전역 단축키를 설정할 수 있다.
- 기본 캡처 방식을 설정할 수 있다.
- 캡처 후 자동 복사를 사용할지 설정할 수 있다.
- 기본 캡처 완료 후 동작을 `저장` 또는 `저장 안 함`로 설정할 수 있다.
- 앱으로 자동 전달 캡처 완료 후 동작을 `전달` 또는 `저장 및 전달`로 설정할 수 있다.
- 앱으로 자동 전달 대상을 `매번 앱 선택`, `가장 최근 사용 앱`, `지정한 앱`으로 설정할 수 있다.
- `지정한 앱`을 선택하면 현재 실행 중인 앱 중 하나를 지정 앱으로 저장할 수 있다.
- 지정 앱의 윈도우 처리를 `가장 최근 사용 윈도우` 또는 `윈도우 선택`으로 설정할 수 있다.
- macOS 시스템 알림 설정으로 이동할 수 있다.
- macOS 시스템 알림이 꺼져 있으면 설정 화면에서 그 상태를 명확히 보여줘야 한다.
- Screen Recording, Accessibility, Notifications 권한 상태를 확인할 수 있다.
