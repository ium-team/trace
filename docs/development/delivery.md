# 앱 자동 전달 정책

## 목표

MVP의 자동 전달은 모든 앱에 대한 완전한 네이티브 연동이 아니라, 캡처 이미지를 클립보드에 복사한 뒤 대상 앱에 붙여넣기 이벤트를 보내는 방식으로 구현한다.

## 기본 방식

자동 전달 순서:

1. 캡처 이미지를 저장한다.
2. 캡처 이미지를 `NSPasteboard`에 복사한다.
3. 실행 중인 앱 목록을 보여준다.
4. 사용자가 대상 앱을 선택한다.
5. Trace가 대상 앱을 활성화한다.
6. Trace가 붙여넣기 이벤트를 보낸다.
7. 전달 결과를 히스토리에 기록한다.

## 대상 앱 목록

대상 앱 목록은 `NSWorkspace`의 실행 중인 앱 목록을 기준으로 만든다.

표시 기준:

- 앱 이름이 있는 항목만 표시한다.
- 백그라운드 전용 프로세스는 제외한다.
- Trace 자신은 제외한다.
- 앱 아이콘과 앱 이름을 함께 표시한다.
- 현재 활성 앱은 우선 표시할 수 있다.

## 권한

자동 전달에는 Accessibility 권한이 필요하다. 권한이 없으면 대상 앱 활성화와 붙여넣기 이벤트 전송을 시도하지 않는다.

권한이 없을 때의 동작:

- 저장은 유지한다.
- 클립보드 복사는 유지한다.
- 사용자에게 권한 안내를 표시한다.
- `deliveryState`는 `failed`로 기록한다.

## 결과 상태

`deliveryState` 값:

- `none`: 복사만 하기 모드
- `skipped`: 사용자가 대상 앱 선택을 건너뜀
- `delivered`: 대상 앱 활성화와 붙여넣기 이벤트 전송을 완료함
- `failed`: 권한 부족, 앱 활성화 실패, 이벤트 전송 실패

`delivered`는 붙여넣기 이벤트를 성공적으로 보냈다는 뜻이다. 대상 앱 내부에서 이미지가 실제로 첨부되었는지까지 100% 보장하지 않는다.

## 실패 처리

실패로 기록하는 경우:

- Accessibility 권한이 없음
- 대상 앱이 이미 종료됨
- 대상 앱 활성화 실패
- 붙여넣기 이벤트 전송 실패

실패 시 사용자에게 제공할 행동:

- 직접 붙여넣기 안내
- Finder에서 파일 보기
- 권한 설정 열기

## MVP 검증 대상 앱

MVP에서는 대표 앱에서 붙여넣기 기반 전달을 검증한다.

우선 대상:

- Slack
- Notion
- TextEdit
- Mail

대상 앱마다 붙여넣기 수용 방식이 다를 수 있으므로, 자동 전달은 앱별 커스텀 규칙 없이 best-effort 방식으로 다룬다.

## MVP 이후 확장

- 자주 쓰는 앱 즐겨찾기
- 앱별 전달 규칙
- 특정 앱의 첨부 API 또는 URL scheme 연동
- 전달 성공 여부를 앱별로 더 정확히 확인하는 방식
