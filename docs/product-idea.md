# Trace 제품 아이디어 문서

## 개요

Trace는 macOS 기본 캡처 경험의 불편함을 줄이기 위한 빠른 캡처 도구다. 사용자는 어디서든 캡처를 시작하고, 원하는 영역을 빠르게 저장한 뒤, 상황에 맞는 캡처 방식을 사용할 수 있다. 단순히 클립보드에 자동 복사만 할 수도 있고, 자동 전달 캡처 상태에서 영역을 선택한 뒤 현재 실행 중인 앱 중 하나를 골라 캡처 이미지를 자동으로 전달할 수도 있다. 캡처 결과는 날짜별 폴더에 저장되며, 앱 안에서 최근 캡처와 과거 캡처 이력을 쉽게 다시 찾을 수 있다.

장기적으로는 AI가 캡처 내용을 분석해 파일명, 폴더, 태그를 자동으로 정리하는 방향까지 확장한다. 다만 MVP에서는 AI 기능을 제외하고, 빠른 캡처, 저장, 앱 전달, 히스토리 관리에 집중한다.

## 해결하려는 문제

macOS 기본 캡처는 강력하지만, 반복적으로 캡처를 다루는 사용자에게는 다음 문제가 있다.

- 캡처 후 결과물을 어디에 저장했는지 추적하기 어렵다.
- 캡처 이미지를 다른 앱에 옮기는 과정이 번거롭다.
- 최근 캡처를 다시 찾거나 재사용하는 흐름이 약하다.
- 캡처 파일이 쌓일수록 정리와 검색 비용이 커진다.
- 사용자가 캡처 후 해야 하는 반복 작업이 많다.

Trace는 캡처 이후의 흐름까지 제품 안에 포함해, 캡처를 단순 저장이 아니라 "획득, 전달, 보관, 재사용"의 작업 단위로 다룬다.

## 핵심 가치

- 빠른 실행: 어떤 앱을 사용 중이든 단축키로 즉시 캡처를 시작한다.
- 빠른 전달: 자동 전달 캡처 모드에서는 캡처 범위 선택 직후 현재 실행 중인 앱 중 전달 대상을 선택하면 이미지가 해당 앱으로 자동 전달된다.
- 자동 복사: 복사만 하기 모드에서는 캡처 완료 후 클립보드에 이미지가 자동 복사된다.
- 선택 가능한 흐름: 사용자는 상황에 따라 복사만 할지, 앱으로 자동 전달할지 고를 수 있다.
- 자동 저장: 사용자가 지정한 기본 위치에 날짜별 폴더로 저장된다.
- 쉬운 재사용: 최근 캡처와 과거 캡처 이력을 앱 안에서 빠르게 확인한다.
- 확장 가능한 정리: 향후 AI 기반 자동 분류와 이름 지정으로 발전할 수 있다.

## 대상 사용자

### 1. 반복적으로 화면을 공유하는 사용자

개발자, 디자이너, 기획자, QA처럼 이슈 공유, 피드백, 문서화 과정에서 화면 캡처를 자주 사용하는 사람이다. 캡처한 뒤 Slack, Notion, Jira, Linear, Figma, 메일 등에 옮기는 일이 많다.

### 2. 캡처 자료를 나중에 다시 찾아야 하는 사용자

업무 기록, 버그 재현, 레퍼런스 수집, 강의나 회의 자료 정리를 위해 과거 캡처를 다시 찾아야 하는 사람이다. 단순히 바탕화면에 파일이 쌓이는 방식보다 히스토리 중심의 접근이 필요하다.

### 3. 기본 캡처 도구보다 한 단계 빠른 흐름을 원하는 사용자

파일 저장, 클립보드 복사, 앱 전환, 붙여넣기를 반복하는 사용자는 Trace의 복사만 하기 흐름과 자동 전달 흐름에서 가장 큰 효용을 얻는다.

## 제품 원칙

- 캡처 시작까지의 마찰을 최소화한다.
- 캡처 이후 사용자가 할 행동을 제품이 자연스럽게 이어준다.
- 저장은 항상 예측 가능해야 한다.
- MVP에서는 복잡한 분류보다 안정적인 히스토리와 날짜별 저장을 우선한다.
- AI는 핵심 흐름이 안정화된 뒤 추가한다.
- macOS 네이티브 앱처럼 가볍고 방해가 적어야 한다.

## 장기 제품 방향

MVP 이후 Trace는 단순 캡처 도구에서 캡처 기반 작업 관리 도구로 확장할 수 있다.

- AI 기반 캡처 내용 분석
- 자동 파일명 생성
- 캡처 내용 기반 폴더 추천
- 텍스트 OCR 및 검색
- 앱별 전달 자동화 규칙
- 자주 쓰는 대상 앱 즐겨찾기
- 캡처 후 주석, 하이라이트, 모자이크
- 팀 공유 또는 클라우드 동기화
