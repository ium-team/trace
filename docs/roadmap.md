목표:

- MVP 범위 확정

## 1단계: macOS 앱 기반 만들기

목표:

- macOS에서 실행되는 기본 앱을 만든다.
- 메뉴바 또는 백그라운드 실행 구조를 만든다.
- 설정 저장 구조를 만든다.

핵심 작업:

- 앱 프로젝트 생성
- 메뉴바 아이콘 구성
- 설정 창 구현
- 저장 위치 설정
- 전역 단축키 등록
- 기본 캡처 방식 설정

완료 기준:

- 앱이 백그라운드에서 실행된다.
- 메뉴바에서 캡처, 히스토리, 설정을 열 수 있다.
- 사용자가 저장 위치와 단축키를 설정할 수 있다.

## 2단계: 캡처 기능 구현

목표:

- 전역 단축키로 Trace 캡처 UI를 열고 영역 캡처를 수행한다.

핵심 작업:

- 화면 오버레이 구현
- 영역 선택 UI 구현
- 선택 영역 이미지 캡처
- PNG 저장
- 클립보드 복사
- 저장 완료 알림
- 기본 캡처 모드 구현

완료 기준:

- 사용자가 단축키로 캡처를 시작할 수 있다.
- 선택한 영역이 이미지로 저장된다.
- 캡처 이미지가 클립보드에 복사된다.
- 날짜별 폴더에 파일이 생성된다.
- 기본 캡처 모드에서는 대상 앱 선택 없이 흐름이 끝난다.

## 3단계: 앱으로 자동 전달 캡처 기능

목표:

- 앱으로 자동 전달 캡처 상태에서 범위를 선택하면 전달 대상 앱을 고르고 붙여넣기 기반 전달을 시도하게 한다.

핵심 작업:

- 실행 중인 앱 목록 조회
- 전달 대상 선택 창 구현
- 앱 아이콘과 이름 표시
- 캡처 방식에 따른 분기 처리
- 선택한 앱 활성화
- 붙여넣기 이벤트 전송
- 건너뛰기 동작 구현

완료 기준:

- 앱으로 자동 전달 캡처에서 범위를 선택하면 대상 앱 목록이 표시된다.
- 사용자가 앱을 선택하면 앱 활성화와 붙여넣기 이벤트 전송이 수행된다.

## 4단계: 히스토리 기능

목표:

- 최근 캡처와 과거 캡처를 앱 안에서 확인하고 재사용할 수 있게 한다.

핵심 작업:

- 캡처 메타데이터 저장
- 썸네일 생성
- 최근 캡처 목록 구현
- 날짜별 히스토리 화면 구현
- 이미지 미리보기
- 다시 클립보드 복사
- Finder에서 보기

완료 기준:

- 최근 캡처가 시간 역순으로 표시된다.
- 날짜별 캡처 이력을 확인할 수 있다.
- 사용자가 과거 이미지를 다시 복사하거나 Finder에서 열 수 있다.

## 5단계: MVP 안정화

목표:

- 실제 사용 가능한 수준으로 품질을 올린다.

핵심 작업:

- 캡처 실패 처리
- 저장 실패 처리
- 클립보드 실패 처리
- 자동 전달 실패 처리
- UI 반응 속도 개선
- 기본 사용 시나리오 테스트

완료 기준:

- 실패 상황에서 앱이 멈추지 않는다.
- 핵심 흐름이 반복 사용에 충분히 빠르고 안정적이다.

## MVP 이후

MVP 이후에는 AI 기능과 자동화 수준을 점진적으로 높인다.

우선순위 후보:

- OCR 텍스트 추출
- AI 기반 파일명 추천
- AI 기반 날짜 외 폴더 분류
- 캡처 내용 검색
- 앱별 전달 규칙
- 민감 정보 자동 블러
